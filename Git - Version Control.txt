1. Установка.
	Скачать дистрибутив: https://git-scm.com/
2. Конфигурация.
	В Git существует три места, где хранятся настройки:

	на уровне системы;			\Program Files\Git\etc\gitconfig	// для всех пользователей системы		git config --system
	на уровне пользователя;			\Пользователи\User\.gitconfig		// для конкретного пользователя системы		git config --global
	на уровне проекта (репозитория).	папка_с_проектом\.git\config		// для проекта					git config
3. Имя пользователя и эл. почта.
	Используются для всех коммитов.
	Команда:
	$ git config --global user.name "yasinsky_vs"
	$ git config --global user.email yasinsky93@gmail.com
4. Посмотреть конфиг:
	$ git config --list // весь конфиг
	$ git config user.name // конкретный конфиг
5. Создание репозитория:
	$ git init
6. Клонирование репозитория:
	$ git clone url
	Что происходит?
	Команда git clone git://github.com/yasinsky93/activebox создает папку activebox, инициализирует в ней папку git,
	клонирует в activebox все файлы проекта.
	Если нужно клонировать файл в папку с отличным названием, нужно добавить ее имя в конце команды:
	git clone git://github.com/yasinsky93/activebox NEWFOLDER
7. Отслеживание новых файлов:
	1. Создаем новый файл (пример - readme.txt). В этот момент в git status он будет в статусе "Untracked files"
	2. Отслеживанием файл командой git add readme.txt. Теперь git status - new file. Коммит выполнит версию файла на момент git add
	3. Если в файл внести изменения, но не выполнить git add, то произойдет коммит той версии, который выполнен git add.
	4. Если создана новая папка, можно добавить к отслеживанию новую папку git add newfolder. Тогда, все файлы в папке будут
	отслеживаться.
	5. Чтобы добавить конкретный файл в папке, нужно сначала перейти в эту папку через cd, а затем добавить конкретный файл.
8. Перемещение между папками.
	ls - посмотреть файлы в текущей папке
	cd folder - переместиться в папку folder
	cd .. - вернуться на папку назад
9. Коммит
	git commit -m 'message'
	git commit -am 'message' // Производит коммит с автоматическим add всех ранее индексированных файлов
10. Удаление файлов
	Удалить с гитхаб и оставить локально:
		1. Прекращаем отслеживание файла: git rm --cached NAME
		2. Выполняем commit изменений
		3. Выполняем push
		4. Файл удален с GitHub, локально остался существовать на винчестере
	Удалить с гитхаб и с ПК:
		1. git rm NAME // Файл удален локально
		2. git commit
		3. git push 
		4. Файл удален локально и с GitHub
	Удалять можно файлы, каталоги и шаблоны.
11. Игнорирование файлов.
	Для игнорирования файлов нужно создать файл .gitignore (без расширения). В файле в каждой строке нужно указывать имя игнорируемого файла.
	Либо использовать шаблонные строки для игнорирования по шаблону.
	Чтобы игнорировать папку - перед ее названием нужно добавить /. Все файлы внутри папки будут проигнорированы.
12. Перемещение / переименование файлов
	git mv OLDNAME NEWNAME // Мы переносим файл в этот же каталог, но с другим имененем
	git mv OLDNAME /newfolded/OLDNAME // Мы переносим файл в каталог newfolder с этим же именем
	git mv OLDNAME /newfolded/NEWNAME // Мы переносим файл в каталог newfolder с именем NEWNAME		
13. Логирование
	git log // посмотреть историю изменений
	git log -2 // ограничить историю изменений последними 2 записями
	git log -p -2 // показать историю 2 последних изменений с diff
	$ git log --since=2.weeks // история изменений за последние 2 недели
14. Изменение коммитов:
	1. Изменение последнего коммита:
		$ git commit -m 'initial commit'
		$ git add forgotten_file
		$ git commit --amend
	2. Отмена индексации файла:
		Если файл случайно проиндексирован (git add), но не нужен в коммите, нужно выполнить $ git reset HEAD NAME. Он будет удален из (git add).
	3. Вернуть файл к состоянию последнего коммита: $ git checkout -- NAME
12. Ветки
	Показать текущую ветку - git branch
	Создать ветку - git branch NAME
	Удалить ветку - git branch -D NAME
	Переключиться на ветку - git checkout NAME
	Сразу создать и переключиться на ветку: git checkout -b NAME
	Объединение веток: git merge NAME - объединяет текущую ветку с веткой NAME
12. Связать репозиторий с гитхаб:
	git remote add origin URL
13. Просмотрудаленных репозиториев:
	git remote -v
13. Залить на гитхаб:
	git push 
14. Просмотр изменений
	git diff - показывает внесенные НЕпроиндексированные изменения (modified)
	git diff --cached - показывает внесенные проиндексированные изменения (modified + added, but not commited)
15. Ветвление
	git branch NAME // создать ветку NAME
	git checkout NAME // переключиться на ветку NAME
	git branch -d NAME // удалить ветку NAME
16. Откат изменений
	git checkout [cash commit]
99. Полезные штуки
	clear - очистить консоль
	ls либо dir - список файлов и папок в текущей директории
	mkdir NAME - создать папку NAME
	git log - история commit-ов
	git show - показать информацию
	git show [хэш коммита] - показать детальную информацию о коммите
	git restore [file]- откатывает изменения в файле [file] до последнего коммита
	HEAD - указывает на коммит, по состоянию на который должны быть файлы в рабочем (локальном) каталоге
